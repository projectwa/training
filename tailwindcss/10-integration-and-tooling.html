10. Integration & Tooling (Tailwind CSS)

Last updated: 2025-10-18 16:02 IST

This chapter shows how to integrate Tailwind CSS into different toolchains and supercharge your workflow with plugins, linters, IDE helpers, testing, and CI/CD. It’s practical and copy‑paste friendly.

1) Quick Start Paths

A) Play CDN (fast prototypes)

<!doctype html>
<html>
<head>
  <script src="https://cdn.tailwindcss.com"></script>
</head>
<body class="min-h-screen grid place-items-center bg-gray-50">
  <button class="rounded-lg bg-indigo-600 px-4 py-2 text-white hover:bg-indigo-500 focus:outline-none focus:ring-2 focus:ring-indigo-300">
    Hello Tailwind
  </button>
</body>
</html>

Pros: zero setup. Cons: no tree‑shaking, limited config.

B) CLI + PostCSS (recommended)

npm i -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
/* src/input.css */
@tailwind base;
@tailwind components;
@tailwind utilities;
// tailwind.config.js
export default {{
  content: ["./index.html", "./src/**/*.{js,ts,jsx,tsx,vue,svelte}"],
  theme: {{ extend: {{}} }},
  plugins: [],
}};
// package.json
{{
  "scripts": {{
    "dev": "npx @tailwindcss/cli -i ./src/input.css -o ./public/tailwind.css --watch",
    "build": "npx @tailwindcss/cli -i ./src/input.css -o ./public/tailwind.css"
  }}
}}

Then include <link href="./public/tailwind.css" rel="stylesheet"> in your HTML.

C) Vite (React/Vue/Svelte/Vanilla)

npm create vite@latest my-app -- --template react-ts
cd my-app
npm i -D tailwindcss postcss autoprefixer
npx tailwindcss init -p

2) Framework Recipes

Next.js (App Router)

import "./globals.css";
export default function RootLayout({{ children }}) {{
  return <html lang="en"><body className="bg-white text-gray-900">{{children}}</body></html>;
}}

Vue (Vite)

Import src/style.css with the @tailwind directives in main.ts. Ensure content globs include Vue files and index.html.

SvelteKit

Add @tailwind to src/app.css, import in +layout.svelte, and include Svelte file globs.

Laravel (Vite)

php artisan breeze:install # optional
npm i -D tailwindcss postcss autoprefixer
npx tailwindcss init -p

Rails

With esbuild or importmap; include ERB/HAML/SLIM templates in content globs.

3) Official Plugins

npm i -D @tailwindcss/typography @tailwindcss/forms @tailwindcss/aspect-ratio @tailwindcss/container-queries
plugins: [
  require("@tailwindcss/typography"),
  require("@tailwindcss/forms"),
  require("@tailwindcss/aspect-ratio"),
  require("@tailwindcss/container-queries"),
]

4) IDE & DX Boosters

  • Tailwind CSS IntelliSense (VS Code)
  • Prettier + prettier-plugin-tailwindcss for class sorting
  • ESLint + eslint-plugin-tailwindcss

5) Storybook Integration

npx storybook@latest init
import "../src/index.css"; // .storybook/preview.js

6) Testing Workflows

await expect(page.getByRole("button", {{ name: "Submit" }})).toHaveCSS("border-radius", "8px");

7) Performance & Production

  • Accurate content globs for tree‑shaking
  • safelist dynamic classes from CMS/user input
  • Use darkMode: "class" for cache‑friendly theming
  • Enable compression and long‑term caching in production

8) CI/CD Tips

  • Cache npm and bundler caches
  • Use npm ci for reproducible installs
  • Run format/lint checks and build

9) Debugging Cheats

/* dev-only.css */
*:not(path):not(svg) {{ outline: 1px dashed rgba(0,0,0,.1); outline-offset: 2px; }}

Try focus utilities like focus:ring-2 focus:ring-blue-300 to verify focus states.
