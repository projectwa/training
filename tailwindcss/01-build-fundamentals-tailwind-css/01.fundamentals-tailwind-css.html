How to use this demo

This page showcases Tailwind's core ideas with simple, workable examples. Each block includes a short explanation and the HTML snippet you can copy. Tailwind is included via the cdn.tailwindcss.com script tag.

1) Utility-first basics

Compose small classes directly on elements—no separate CSS file required.

Welcome!

Tailwind makes styling easy and fast.

<div class="bg-gray-100 p-6 rounded-lg shadow-inner">
  <h3 class="text-2xl font-bold text-gray-800 mb-2">Welcome!</h3>
  <p class="text-gray-600">Tailwind makes styling easy and fast.</p>
</div>

2) Responsive design

Tailwind is mobile-first. Unprefixed classes apply to all sizes, and breakpoint-prefixed classes (like sm:, md:, lg:) override styles at that width and above.

Resize the window to see this background color change at breakpoints. 
<div class="p-4 text-white text-center
  bg-green-500 sm:bg-blue-500 md:bg-yellow-500 lg:bg-pink-500">...</div>

3) States: hover & focus

Use state variants to style interactions like hover and focus. Accessible focus rings are easy to add. 

<button class="bg-blue-600 hover:bg-blue-700
  focus:ring-2 focus:ring-blue-300 text-white font-semibold px-6 py-3 rounded">...</button>

5) Transitions & transforms

Add delightful micro-interactions without custom CSS.

<button class="px-4 py-2 bg-purple-600 text-white rounded
  transition-transform duration-300 hover:scale-105">...</button>

8) Customization

With an NPM setup, extend your design tokens in tailwind.config.js. This block simulates a "brand" color.

Custom brand color (example)
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: { brand: "#1e40af" }
    }
  }
};
