# 2) Tailwind CSS — Core Concepts (with workable examples)

This guide explains Tailwind’s core ideas with examples you can run. Use the **HTML demo file** included with this guide to see most concepts working immediately (uses the Tailwind CDN).

> If you’re using the CLI/build workflow, you’ll get **all features** (like `@apply`, custom config, and purging). The CDN is perfect for quick demos but not for production.

---

## 1. Utility‑first styling

Tailwind gives you **small, single‑purpose classes** (utilities). Compose them on your HTML elements instead of writing CSS selectors.

```html
<button class="px-4 py-2 bg-blue-600 text-white rounded-lg hover:bg-blue-700">
  Save
</button>
```

### Why?
- **Fast**: Style while you build UI.
- **Predictable**: Utilities map directly to CSS.
- **Low context switching**: Stay in your markup.

---

## 2. Responsive design (mobile‑first)

Tailwind uses **breakpoint prefixes** like `sm: md: lg: xl: 2xl:` to change styles at specific widths. Styles **without** a prefix apply to mobile first, then you enhance upwards.

```html
<div class="p-4 bg-white sm:bg-emerald-50 md:bg-yellow-50 lg:bg-blue-50">
  Resize the viewport to see the background color change
</div>
```

**Default breakpoints** (can be customized):
- `sm` 640px
- `md` 768px
- `lg` 1024px
- `xl` 1280px
- `2xl` 1536px

---

## 3. State & pseudo‑class variants

Variants let you style for **interaction states** like hover, focus, active, disabled, first/last child, etc.

```html
<input
  class="border rounded px-3 py-2 focus:outline-none focus:ring-2 focus:ring-blue-400"
  placeholder="Focus me to see a ring" />
```

Common variants: `hover:`, `focus:`, `active:`, `disabled:`, `visited:`, `first:`, `last:`, `odd:`, `even:`, `aria-*`, `peer-*`.

---

## 4. Dark mode

Two modes:
1) **Class strategy** (recommended): Put `class="dark"` on a parent to enable dark variants.
2) **Media strategy**: Match `prefers-color-scheme`.

```html
<div class="p-4 bg-white text-gray-900 dark:bg-gray-900 dark:text-gray-100">
  Works when a parent has the <code>dark</code> class or when using media strategy.
</div>
```

> In CDN demos, toggle the `dark` class on `<html>` or a wrapper to preview.

---

## 5. Layout with Flexbox & Grid

**Flexbox**
```html
<div class="flex items-center justify-between gap-4">
  <span>Left</span>
  <span>Center grows</span>
  <span class="ml-auto">Right</span>
</div>
```

**Grid**
```html
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4">
  <div class="p-4 bg-white rounded shadow">Card 1</div>
  <div class="p-4 bg-white rounded shadow">Card 2</div>
  <div class="p-4 bg-white rounded shadow">Card 3</div>
</div>
```

---

## 6. Spacing, sizing, and typography scales

Tailwind ships with a design system of **scales**:
- Spacing: `p-4`, `m-2`, `space-x-3`, etc.
- Sizing: `w-64`, `h-10`, `max-w-md`, `aspect-video`
- Typography: `text-sm`, `text-xl`, `font-semibold`, `leading-7`, `tracking-wide`

```html
<p class="max-w-prose text-gray-700 text-lg leading-7">
  Prose text with sensible line-height and width for readability.
</p>
```

---

## 7. Transitions, transforms, and animation

```html
<button
  class="px-4 py-2 rounded-lg bg-indigo-600 text-white transition transform hover:-translate-y-0.5 hover:scale-[1.02] hover:bg-indigo-700">
  Hover me
</button>
```

- `transition` enables defaults; fine‑tune with `duration-300`, `ease-in-out`, `delay-150`.
- Transforms: `scale-…`, `rotate-…`, `translate-…`, `skew-…`.

---

## 8. Composing utilities with `@apply` (build workflow)

Use `@apply` inside your CSS **when using the CLI/PostCSS build** (not supported via CDN).

```css
/* input.css */
@tailwind base;
@tailwind components;
@tailwind utilities;

.btn-primary {
  @apply px-4 py-2 rounded-lg bg-blue-600 text-white hover:bg-blue-700;
}
```

```html
<button class="btn-primary">Save</button>
```

---

## 9. Arbitrary values & arbitrary variants

When a utility doesn’t exist, use an **arbitrary value**:

```html
<div class="h-[72px] w-[22rem] bg-[rebeccapurple]"></div>
```

Arbitrary variants target things like complex selectors or media queries (build workflow). Example with `supports-[...]`:

```html
<div class="supports-[backdrop-filter]:backdrop-blur-md">
  This blurs if the browser supports backdrop-filter.
</div>
```

---

## 10. Reusable components with `@layer` (build workflow)

Organize custom components/utilities in `@layer components` or `@layer utilities` so Tailwind can **merge and purge** correctly.

```css
@layer components {
  .card {
    @apply rounded-2xl shadow p-6 bg-white dark:bg-gray-900;
  }
  .card-title {
    @apply text-lg font-semibold mb-2;
  }
}
```

## 11. Customizing the design system (tailwind.config.js)

You can extend colors, spacing, fonts, etc.

```js
// tailwind.config.js
module.exports = {
  darkMode: 'class',
  content: ['./public/**/*.html'],
  theme: {
    extend: {
      colors: {
        brand: {
          600: '#5B21B6',
          700: '#4C1D95',
        },
      },
      fontFamily: {
        display: ['Inter', 'system-ui', 'sans-serif'],
      },
    },
  },
  plugins: [],
};
```

Then use: `bg-brand-600`, `font-display`.

## 12. Forms, typography, and other plugins

Official plugins like `@tailwindcss/forms`, `@tailwindcss/typography`, and `@tailwindcss/aspect-ratio` add polished defaults.

```bash
npm i -D @tailwindcss/forms @tailwindcss/typography @tailwindcss/aspect-ratio
```

```js
// tailwind.config.js
plugins: [
  require('@tailwindcss/forms'),
  require('@tailwindcss/typography'),
  require('@tailwindcss/aspect-ratio'),
]
```

---

## 13. Production build (purge)

Keep CSS tiny by scanning your content files and removing unused classes (handled by Tailwind’s `content` option + CLI).

```json
// package.json scripts
{
  "scripts": {
    "build": "npx @tailwindcss/cli -i ./src/input.css -o ./public/tailwind.css"
  }
}
```

Make sure `tailwind.config.js` has a `content` array listing your templates (e.g., `./public/**/*.html`).

---

## 14. Accessibility helpers

- Focus styles: `focus:outline-none focus:ring-2 focus:ring-blue-400`
- Visible skip links: `sr-only focus:not-sr-only`
- Use `aria-*` variants: `aria-selected:bg-blue-600`

```html
<a href="#main" class="sr-only focus:not-sr-only focus:absolute focus:top-2 focus:left-2">
  Skip to content
</a>
```
