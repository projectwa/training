# 01. Fundamentals of Tailwind CSS

Tailwind CSS is a **utility-first** CSS framework that lets you build modern, responsive UIs quickly by composing small classes directly in your HTML ‚Äî no custom CSS required for most cases.

---

## üåü What is Tailwind CSS?

Traditional CSS approach:

```css
.btn {
  background-color: #3b82f6;
  color: white;
  padding: 1rem 2rem;
  border-radius: 0.5rem;
}
```

**Tailwind approach (no CSS file needed):**

```html
<button class="bg-blue-500 text-white px-8 py-4 rounded-lg">Click Me</button>
```

---

## ‚öôÔ∏è Setting Up Tailwind

You can use Tailwind in **three ways**:

### 1) Quick Start via CDN (best for learning / demos)
Add the script in your HTML `<head>`:
```html
<script src="https://cdn.tailwindcss.com"></script>
```

### 2) NPM + Build (best for production)
```bash
npm install -D tailwindcss
npx tailwindcss init
```
Configure `tailwind.config.js` and run the CLI to generate a production CSS file with **purged** (tree‚Äëshaken) utilities.

### 3) Framework Integrations
Works seamlessly with frameworks like **Next.js**, **Vite**, **Laravel**, etc.

## üß± Utility-First Philosophy

Each class does **one small job** (e.g., margin, padding, color). You **compose** them together to style an element.

```html
<div class="bg-gray-100 p-6 rounded-lg shadow-md">
  <h1 class="text-2xl font-bold text-gray-800 mb-2">Welcome!</h1>
  <p class="text-gray-600">Tailwind makes styling easy and fast.</p>
</div>
```

**Why this is powerful:** faster dev speed, consistent design system, avoids naming CSS classes, and it scales well with components.

## üé® Common Utility Categories

| Category      | Examples                                   | Description                          |
|---------------|---------------------------------------------|--------------------------------------|
| Colors        | `bg-blue-500`, `text-red-600`               | Background & text colors             |
| Spacing       | `m-4`, `p-2`, `px-6`, `py-3`                | Margin & padding                     |
| Typography    | `text-lg`, `font-bold`, `tracking-wide`     | Font size, weight, letter spacing    |
| Layout        | `flex`, `grid`, `items-center`, `justify-between` | Flexbox & grid utilities        |
| Borders       | `border`, `border-gray-300`, `rounded-lg`   | Border width, color, radius          |
| Shadows       | `shadow`, `shadow-lg`, `shadow-inner`       | Box shadows                          |
| Sizing        | `w-1/2`, `h-64`, `max-w-md`                 | Width & height                       |
| Effects       | `transition`, `duration-300`, `opacity-75`  | Transitions & animations             |

## üì± Responsive Design (Mobile-First)

Use breakpoint prefixes: `sm:`, `md:`, `lg:`, `xl:`, `2xl:`

```html
<div class="bg-green-400 p-4 sm:bg-blue-400 md:bg-yellow-400 lg:bg-pink-400">
  Resize the screen to see color change!
</div>
```

**Default breakpoints:**

| Prefix | Min Width |
|-------:|-----------|
| `sm:`  | 640px     |
| `md:`  | 768px     |
| `lg:`  | 1024px    |
| `xl:`  | 1280px    |
| `2xl:` | 1536px    |

> Tailwind is mobile-first: unprefixed classes apply to all sizes, then **prefixed** ones override at that breakpoint and above.

## üåà State Variants (hover, focus, active, etc.)

```html
<button class="bg-blue-500 hover:bg-blue-600 focus:ring-2 focus:ring-blue-300 text-white font-semibold px-6 py-3 rounded">
  Hover Me
</button>
```

- `hover:bg-blue-600` ‚Üí background on hover  
- `focus:ring-2 focus:ring-blue-300` ‚Üí accessible focus ring

Other common variants: `active:`, `disabled:`, `focus-visible:`, `aria-*`, `data-*`.

## ‚öíÔ∏è Customization with `tailwind.config.js`

Extend theme tokens (colors, spacing, fonts, etc.).

```js
// tailwind.config.js
module.exports = {
  content: ["./src/**/*.{html,js,ts,tsx}"],
  theme: {
    extend: {
      colors: {
        brand: "#1e40af"
      },
    },
  },
  plugins: [],
};
```

Usage:

```html
<div class="bg-brand text-white p-4">Custom color example</div>
```

## üß† Tips, Patterns & Best Practices

- **Start with CDN** for learning; switch to **NPM build** for production to enable purging and faster CSS.  
- Prefer **composing utilities** over writing custom CSS. If repeated often, extract a component or add **`@apply`** in a CSS layer (NPM setup).  
- Keep class names readable by grouping by concern (layout ‚Üí spacing ‚Üí color ‚Üí effects).  
- Use **responsive & state variants** thoughtfully: `md:flex lg:items-center hover:bg-gray-50`.  
- For design consistency, **extend theme** instead of ad-hoc hex colors everywhere.
