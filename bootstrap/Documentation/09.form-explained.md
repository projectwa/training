# Bootstrap 5 Form – All Fields & Validation (CDN)

This package contains:

- `bootstrap-form.html` — a comprehensive form implemented with **Bootstrap 5 (via CDN)** using `.form-control`, `.form-select`, `.form-check`, and built‑in **validation classes** (`.is-valid`, `.is-invalid`, `.valid-feedback`, `.invalid-feedback`).
- `bootstrap-form-explained.md` (this file) — how it’s structured and how to adapt it.

---

## Why Bootstrap?

- Consistent, responsive UI out of the box
- Standard patterns for forms and validation
- Easy to drop in via **CDN** for demos and prototypes

> The HTML links to Bootstrap **5.3.3** CDN for both CSS and JS bundle.

---

## Structure

- **Grid**: Uses `.row` + `.col-md-*` to lay out fields in two columns on desktop, one on mobile.
- **Labels**: Every input has a `<label>` with `for` matching the input `id`.
- **Help/Hint**: Uses `.form-text` for hints.
- **Groups**: Radios and checkboxes use `.form-check` and are wrapped in a `<fieldset><legend>…</legend></fieldset>`.

Sections included:

1. Basics (name, email, password, phone)  
2. URLs & Search  
3. Numbers & Ranges  
4. Dates & Time  
5. Choices (selects, multi-select, radios, checkboxes)  
6. Text & Files (textarea, file inputs)  
7. Hidden/readonly example

## Validation

This template combines **native HTML5 constraints** with Bootstrap’s validation styling:

- Required fields: `required`
- Length & numeric constraints: `minlength`, `maxlength`, `min`, `max`, `step`
- Patterns: e.g. password need **1 digit** via `pattern="(?=.*\d).{8,}"`
- File validation: example custom rule for **avatar** (≤ 2 MB per file) via `setCustomValidity`

### How it works

- The `<form>` has `novalidate` and the class `needs-validation` (a common Bootstrap pattern).
- On **input/blur**, we call `checkValidity()` and toggle `.is-valid` / `.is-invalid`.
- On **submit**, we:
  - validate all controls (including radio groups by `name`)
  - build an **error summary** in a Bootstrap `.alert-danger` at the top
  - prevent submission for the demo and show a success **Toast** if all good

To actually submit, replace the submit handler with your own `fetch()` or standard submit.

```js
form.addEventListener('submit', async (e) => {
  e.preventDefault();
  if (!validateForm()) return;
  const data = new FormData(form);
  const res = await fetch('/submit', { method: 'POST', body: data });
  // handle response...
});
```

---

## Customization Tips

- Toggle light/dark via `<html data-bs-theme="light|dark">`.
- Replace `.form-section-title` styling or switch to Bootstrap utilities (e.g., `border-bottom`, `pb-1`, `mt-4`).
- If using React, keep the same semantics; apply `.is-valid`/`.is-invalid` based on component state.
