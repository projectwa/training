# 02 ‚Äì What Is a Mixin in Sass?

A **mixin** in **Sass** is a reusable block of CSS code that you can define once and include anywhere in your stylesheets.  

Think of it like a **function in programming** ‚Äî it can take parameters and inject styles wherever you need them.

---

## üß† Why Use Mixins?

Mixins help you:
- Avoid writing repetitive CSS
- Keep your styles **DRY** (Don‚Äôt Repeat Yourself)
- Add flexibility with **parameters** and **default values**
- Generate **vendor-prefixed properties** easily (e.g., for animations, transforms)

---

## üß© Basic Example

```scss
@mixin rounded {
  border-radius: 10px;
  border: 1px solid #ccc;
}

.button {
  @include rounded;
  background-color: #007bff;
  color: white;
}
```

**Explanation:**
- `@mixin rounded` defines a reusable style block.
- `@include rounded` inserts the mixin‚Äôs content into `.button`.

The compiled CSS becomes:

```css
.button {
  border-radius: 10px;
  border: 1px solid #ccc;
  background-color: #007bff;
  color: white;
}
```

---

## ‚öôÔ∏è Mixins with Parameters

Mixins can take arguments to make them flexible:

```scss
@mixin box($width, $height, $color) {
  width: $width;
  height: $height;
  background-color: $color;
}

.card {
  @include box(200px, 100px, lightblue);
}
```

Output CSS:

```css
.card {
  width: 200px;
  height: 100px;
  background-color: lightblue;
}
```

---

## üé® Mixins with Default Values

You can define default parameters that can be overridden when needed:

```scss
@mixin text-style($size: 16px, $color: #333) {
  font-size: $size;
  color: $color;
}

p {
  @include text-style; // Uses defaults
}

h1 {
  @include text-style(32px, #000); // Overrides defaults
}
```

---

## ‚ö° Common Real-World Example: Vendor Prefixes

Instead of writing multiple vendor-prefixed properties manually, use a mixin:

```scss
@mixin transform($value) {
  -webkit-transform: $value;
  -ms-transform: $value;
  transform: $value;
}

.box {
  @include transform(rotate(45deg));
}
```

---

## ‚úÖ Summary

| Concept | Description |
|----------|--------------|
| **@mixin** | Defines a reusable block of styles |
| **@include** | Inserts a mixin where it's needed |
| **Parameters** | Allow flexibility and reuse with different values |
| **Default values** | Provide fallbacks for optional arguments |

Mixins make your Sass code **modular, readable, and efficient**, especially in large-scale projects like Bootstrap customization.
