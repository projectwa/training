# 03 ‚Äì Sass Maps and Loops

Sass Maps and Loops are advanced features that make your stylesheets **dynamic**, **modular**, and **maintainable**.

---

## üó∫Ô∏è Sass Maps

A **map** in Sass is similar to an **object** or **dictionary** in programming.  
It stores **key‚Äìvalue pairs**, making it easy to organize and retrieve related data ‚Äî like colors, breakpoints, or theme variables.

### üß© Basic Example

```scss
$colors: (
  "primary": #007bff,
  "secondary": #6c757d,
  "success": #28a745,
  "danger": #dc3545
);

// Access a value
.btn-primary {
  background-color: map-get($colors, "primary");
}
```

**Explanation:**
- `$colors` is a map.
- `map-get($colors, "primary")` retrieves the value `#007bff`.

---

### ‚úèÔ∏è Updating or Adding Map Values

You can add or modify entries using `map-merge()`:

```scss
$colors: map-merge(
  $colors,
  ("warning": #ffc107)
);
```

Now `$colors` includes `"warning"`.

---

### üß† Why Use Maps?

They help you:
- Manage multiple related variables (like color themes, font sizes)
- Make your code scalable
- Reduce redundancy and human error

## üîÅ Sass Loops

Loops in Sass let you **repeat styles automatically** ‚Äî perfect for generating multiple classes from data (like your map).

There are three main types:

---

### 1. @each Loop

Used to iterate over **lists** or **maps**.

```scss
$colors: (
  "primary": #007bff,
  "secondary": #6c757d,
  "success": #28a745,
);

@each $name, $color in $colors {
  .text-#{$name} {
    color: $color;
  }
}
```

**Output:**

```css
.text-primary { color: #007bff; }
.text-secondary { color: #6c757d; }
.text-success { color: #28a745; }
```

‚úÖ Used for: **Dynamic class generation**

---

### 2. @for Loop

Used when you need numeric iteration.

```scss
@for $i from 1 through 3 {
  .m-#{$i} {
    margin: #{$i * 5}px;
  }
}
```

**Output:**

```css
.m-1 { margin: 5px; }
.m-2 { margin: 10px; }
.m-3 { margin: 15px; }
```

‚úÖ Used for: **Spacing utilities**, grid systems, etc.

---

### 3. @while Loop

Runs as long as a condition is true.

```scss
$i: 1;
@while $i <= 3 {
  .p-#{$i} {
    padding: #{$i * 10}px;
  }
  $i: $i + 1;
}
```

**Output:**

```css
.p-1 { padding: 10px; }
.p-2 { padding: 20px; }
.p-3 { padding: 30px; }
```

‚úÖ Used for: **Custom patterns or algorithmic styling**

---

## üöÄ Combining Maps and Loops

You can combine both for powerful dynamic styling:

```scss
$theme-colors: (
  "primary": #007bff,
  "success": #28a745,
  "danger": #dc3545
);

@each $name, $color in $theme-colors {
  .btn-#{$name} {
    background-color: $color;
    color: white;
  }
}
```

**Output:**

```css
.btn-primary { background-color: #007bff; color: white; }
.btn-success { background-color: #28a745; color: white; }
.btn-danger { background-color: #dc3545; color: white; }
```

---

## ‚úÖ Summary

| Concept | Description | Example |
|----------|--------------|----------|
| **Map** | A collection of key‚Äìvalue pairs | `("primary": #007bff)` |
| **map-get()** | Retrieves a value from a map | `map-get($colors, "primary")` |
| **@each** | Loops through maps or lists | `@each $key, $val in $map` |
| **@for** | Loops through a range of numbers | `@for $i from 1 through 3` |
| **@while** | Loops while condition is true | `@while $i <= 3` |

Together, **maps + loops** let you generate CSS utilities dynamically ‚Äî just like **Bootstrap** does for colors, spacings, and grid breakpoints.
